## 🔹 Criterios de evaluación comunes

### 1. Comprensión del modelo de procesamiento distribuido (20 %)

| Nivel                 | Descriptor                                                                                                                       |
| --------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| **Excelente (9–10)**  | Explica con claridad el rol de Spark, Master y Workers, y relaciona el modelo distribuido con problemas reales de escalabilidad. |
| **Notable (7–8)**     | Comprende el modelo general, aunque con explicaciones algo superficiales.                                                        |
| **Suficiente (5–6)**  | Identifica los conceptos básicos, pero sin profundidad ni conexión real.                                                         |
| **Insuficiente (<5)** | Confusión clara sobre qué hace Spark o cómo funciona el clúster.                                                                 |

---

### 2. Uso adecuado de las herramientas (Spark / pandas / formatos) (20 %)

| Nivel            | Descriptor                                                                                               |
| ---------------- | -------------------------------------------------------------------------------------------------------- |
| **Excelente**    | Usa cada herramienta con criterio, justificando por qué se emplea Spark, pandas o Parquet según el caso. |
| **Notable**      | Uso correcto, aunque con justificaciones parciales.                                                      |
| **Suficiente**   | Uso funcional, pero sin justificar decisiones.                                                           |
| **Insuficiente** | Uso mecánico o incorrecto de las herramientas.                                                           |

---

### 3. Análisis crítico y reflexión técnica (25 %)

| Nivel            | Descriptor                                                                     |
| ---------------- | ------------------------------------------------------------------------------ |
| **Excelente**    | Respuestas razonadas, técnicas y conectadas con escenarios reales de Big Data. |
| **Notable**      | Respuestas correctas pero algo genéricas.                                      |
| **Suficiente**   | Respuestas breves o poco elaboradas.                                           |
| **Insuficiente** | Respuestas vagas, copiadas o sin razonamiento.                                 |

---

### 4. Documentación y claridad del informe (15 %)

| Nivel            | Descriptor                                                      |
| ---------------- | --------------------------------------------------------------- |
| **Excelente**    | Documento claro, ordenado, bien estructurado y fácil de seguir. |
| **Notable**      | Documento comprensible con pequeños problemas de organización.  |
| **Suficiente**   | Documento incompleto o desordenado, pero entendible.            |
| **Insuficiente** | Documento confuso o difícil de interpretar.                     |

---

### 5. Gestión de problemas y autonomía (10 %)

| Nivel            | Descriptor                                                               |
| ---------------- | ------------------------------------------------------------------------ |
| **Excelente**    | Identifica problemas técnicos, explica causas y soluciones con criterio. |
| **Notable**      | Describe problemas y soluciones básicas.                                 |
| **Suficiente**   | Enumera problemas sin analizarlos.                                       |
| **Insuficiente** | Oculta problemas o no los reconoce.                                      |

---

### 6. Trabajo en grupo (10 %)

| Nivel            | Descriptor                                              |
| ---------------- | ------------------------------------------------------- |
| **Excelente**    | Reparto equilibrado de tareas y buena coordinación.     |
| **Notable**      | Trabajo conjunto correcto con pequeños desequilibrios.  |
| **Suficiente**   | Reparto poco claro, pero funcional.                     |
| **Insuficiente** | Falta de coordinación o dependencia de un solo miembro. |

## 🧪 Ajustes específicos por laboratorio

### ✔️ Lab 1 — Spark básico

Se valora especialmente:

* Comprensión del modelo Master / Worker.
* Capacidad de explicar **qué se ha hecho**, no solo que “funciona”.

---

### ✔️ Lab 2 — Spark vs pandas

Se valora especialmente:

* La comparación razonada entre herramientas.
* Identificación del **punto de ruptura** de pandas.
* Criterio al elegir una u otra solución.

---

### ✔️ Lab 3 — Parquet y particionado

Se valora especialmente:

* Comprensión del papel del **formato de almacenamiento**.
* Justificación del particionado.
* Capacidad de conectar con **datalakes y explotación de datos**.

---

## 📌 Criterios NO penalizables explícitamente

No se penaliza si está bien explicado:

* No haber usado Plan A (clúster real).
* Equipos con recursos limitados.
* Tiempos de ejecución más largos.
* Ajustes de factor de inflado.

👉 **Se evalúa el razonamiento, no la potencia del portátil**.

---

## 🧭 Nota final para el alumnado (puedes incluirla)

> En estos laboratorios no se evalúa solo que algo funcione,
> sino que entiendas **por qué se hace así** y **cuándo tiene sentido usar cada herramienta**.
