# Construcción de datasets desde fuentes reales  
## Punto de encuentro entre Sistemas de Big Data y Programación de IA

---

## 1. Introducción: del dataset “ya hecho” al dataset construido

Hasta ahora, en muchos ejercicios de **Programación de Inteligencia Artificial (PIA)**, los datasets utilizados por el alumnado han sido **datasets ya existentes**: Kaggle, UCI, datasets académicos o CSV previamente preparados.  
Esto es útil para aprender **modelos**, **EDA** y **evaluación**, pero **no refleja cómo se generan los datos en escenarios reales**.

En la práctica profesional:

- Los datos **no están limpios**
- Los datos **proceden de múltiples fuentes**
- Los datos **llegan en distintos formatos**
- Los datos **crecen en volumen con el tiempo**
- Los datos requieren **procesos de ingesta, integración y control de calidad**

Este es el punto natural donde **Sistemas de Big Data** debe intervenir.

---

## 2. ¿Por qué este es un buen momento para introducir Big Data?

La construcción de datasets reales conecta directamente con las **5 V del Big Data**:

| V | Aplicación al proyecto |
|---|------------------------|
| **Volumen** | APIs que generan miles/millones de registros (tiempo, sensores, logs) |
| **Velocidad** | Datos que se actualizan periódicamente (cada hora, cada día) |
| **Variedad** | JSON, CSV, logs, datos semiestructurados |
| **Veracidad** | Datos incompletos, ruidosos, inconsistentes |
| **Valor** | El dataset final se usará para entrenar modelos de IA |

Si los datos:
- provienen de **una sola fuente**
- son **pequeños**
- y se usan una sola vez  

👉 Big Data **no es necesario**.

Si los datos:
- provienen de **varias fuentes**
- crecen con el tiempo
- deben integrarse y versionarse  

👉 **Big Data es el enfoque correcto**, aunque sea a pequeña escala educativa.

---

## 3. Rol de cada módulo en el proyecto

### 3.1 Sistemas de Big Data (SBD)

En SBD el foco está en el **pipeline de datos**, no en el modelo:

- Selección de **fuentes de datos**
- Ingesta (batch o incremental)
- Almacenamiento eficiente
- Integración y limpieza inicial
- Control de calidad
- Generación del **dataset final**

**Herramientas típicas (SBD):**

- Python (requests, pandas, polars)
- Airbyte (opcional)
- DuckDB / Parquet
- Spark (si el volumen lo justifica)
- MinIO / sistema de ficheros
- Git para versionado del dataset

---

### 3.2 Programación de Inteligencia Artificial (PIA)

En PIA el foco es **qué se hace con el dataset**:

- EDA en profundidad
- Feature engineering
- Selección de variables
- Entrenamiento de modelos
- Evaluación y métricas
- Interpretación de resultados

**PIA asume** que el dataset:
- ya existe
- tiene un formato coherente
- tiene una mínima calidad garantizada

Ese dataset lo proporciona **SBD**.

---

## 4. Arquitectura conceptual del proyecto conjunto

```

Fuentes públicas
(API / CSV / Logs)
↓
Ingesta (SBD)
↓
Almacenamiento intermedio
(Parquet / DuckDB)
↓
Limpieza + integración (SBD)
↓
Dataset final versionado
↓
EDA + ML (PIA)

```

## 5. Fuentes de datos propuestas (datasets reales)

### Variante 1 — Meteorología + calidad del aire (recomendada)

**Fuentes:**
- AEMET OpenData  
  https://opendata.aemet.es/
- OpenAQ (calidad del aire)  
  https://openaq.org/

**Características:**
- Datos temporales
- APIs REST
- JSON semiestructurado
- Ideal para series temporales

**Dataset final ejemplo:**
> Predicción de calidad del aire en función de variables meteorológicas

---

### Variante 2 — Movilidad urbana y tráfico

**Fuentes:**
- NYC Open Data – Traffic  
  https://opendata.cityofnewyork.us/
- Madrid Datos Abiertos (tráfico)  
  https://datos.madrid.es/

**Características:**
- CSV grandes
- Datos geoespaciales
- Eventos y series temporales

**Dataset final ejemplo:**
> Predicción de congestión o clasificación de tramos urbanos

---

### Variante 3 — Logs de servicios digitales (simulados)

**Fuentes:**
- Logs sintéticos generados por scripts
- Estructura tipo Apache/Nginx

**Características:**
- Datos no estructurados
- Alto volumen
- Pensado para ampliaciones (Kibana, Spark)

**Dataset final ejemplo:**
> Clasificación de errores o detección de anomalías

## 6. Propuesta de tarea conjunta SBD + PIA

### Fase 1 — Sistemas de Big Data (SBD)

**Objetivo:** construir el dataset

Tareas:
1. Seleccionar 2 fuentes de datos
2. Implementar ingesta (scripts o Airbyte)
3. Normalizar formatos
4. Controlar calidad (nulos, rangos, duplicados)
5. Almacenar en Parquet
6. Documentar el pipeline

**Entregables SBD:**
- Scripts de ingesta
- Dataset en Parquet
- Documento de calidad
- README técnico

---

### Fase 2 — Programación de IA (PIA)

**Objetivo:** explotar el dataset

Tareas:
1. EDA completo
2. Feature engineering
3. Entrenamiento de modelos
4. Evaluación
5. Conclusiones

**Entregables PIA:**
- Notebook EDA
- Notebook ML
- Informe de resultados

---

### Fase 3 — Evaluación conjunta

**Objetivo:** visión end-to-end del dato

- Defensa conjunta
- Justificación de decisiones
- Relación datos → modelo → resultados

## 8. Conclusión didáctica

Este enfoque permite al alumnado:

- Entender **de dónde vienen los datos**
- Valorar la importancia del **pipeline**
- Ver Big Data como **habilitador de IA**
- Trabajar como lo harían en un entorno profesional real

SBD deja de ser “teoría de herramientas” y pasa a ser **la base del dato** sobre la que se construye la IA.
