# Construcción de datasets a partir de fuentes reales

## De la obtención de datos a Big Data (PIA ↔ SBD)

### 1. Contexto: del dataset “hecho” al dataset “construido”

Hasta ahora, en muchos ejercicios de **Programación de Inteligencia Artificial (PIA)**, los alumnos han trabajado con datasets ya existentes: CSVs de Kaggle, datasets académicos, ficheros tabulares “limpios” o semilimpios.
Esto es útil para aprender **modelado, EDA y evaluación**, pero **oculta una parte crítica del trabajo real con datos**: la **construcción del dataset**.

En contextos profesionales, los datos **no aparecen ya estructurados**. Normalmente:

* Proceden de **fuentes heterogéneas**.
* Llegan en **formatos distintos**.
* Se generan de forma **continua** o **incremental**.
* Tienen **errores, huecos, duplicados y ruido**.
* Crecen en volumen con el tiempo.

Es en este punto donde **Big Data deja de ser una palabra de moda** y se convierte en una **necesidad técnica**.

---

### 2. ¿Cuándo tiene sentido que intervenga Big Data?

No todos los problemas requieren tecnologías Big Data.
Tiene sentido introducir **Sistemas de Big Data** cuando el proceso de construcción del dataset presenta claramente alguna de las **5 V**:

#### Volumen

* Datos históricos acumulados durante años (meteorología, tráfico, ventas).
* Logs de sistemas, sensores, eventos de aplicaciones.
* Descargas periódicas de APIs públicas.

Ejemplo:

> Datos horarios de calidad del aire de varias estaciones durante 5 años → millones de registros.

#### Velocidad

* Datos que se generan de forma continua o casi continua.
* Necesidad de actualizaciones frecuentes del dataset.

Ejemplo:

> Datos meteorológicos cada 10 minutos o eventos de tráfico en tiempo real.

#### Variedad

* Diferentes formatos y estructuras:

  * APIs REST (JSON)
  * Logs (texto semiestructurado)
  * CSV históricos
  * Series temporales
  * Datos geoespaciales

Ejemplo:

> API de AEMET + CSV históricos + metadatos de estaciones.

#### Veracidad

* Datos incompletos, inconsistentes o ruidosos.
* Cambios en los esquemas con el tiempo.
* Valores atípicos o erróneos.

Ejemplo:

> Sensores que fallan, estaciones que dejan de reportar datos.

#### Valor

* El dataset no es el fin, sino el **medio**:

  * Entrenamiento de modelos.
  * Análisis de tendencias.
  * Predicción o clasificación.

Aquí es donde **PIA entra en juego**.

---

### 3. Construir un dataset es un pipeline (no un fichero)

Desde el punto de vista de **Sistemas de Big Data**, construir un dataset implica diseñar un **pipeline de datos**, no solo ejecutar un script.

Un pipeline típico incluye:

1. **Ingesta**

   * APIs REST
   * Ficheros descargados periódicamente
   * Logs
   * Sensores / series temporales

2. **Aterrizaje (raw)**

   * Datos tal como llegan
   * Sin transformar
   * Almacenados para trazabilidad

3. **Procesamiento**

   * Limpieza
   * Normalización
   * Enriquecimiento
   * Unión de fuentes

4. **Persistencia analítica**

   * Formatos eficientes (Parquet)
   * Particionado temporal / geográfico
   * Listo para EDA, BI o ML

5. **Consumo**

   * EDA
   * Dashboards
   * Entrenamiento de modelos (PIA)

Este enfoque **no es necesario** cuando trabajamos con un CSV pequeño, pero **es imprescindible** cuando el dataset se construye y evoluciona en el tiempo.

---

### 4. Herramientas adecuadas (realistas para FP)

Sin irnos a infraestructuras empresariales complejas, el stack puede ser **100% razonable para el aula**:

#### Ingesta

* **Python** (requests, aiohttp)
* **Airbyte OSS** (cuando hay múltiples fuentes o conectores)
* Scripts programados (batch)

#### Almacenamiento

* **MinIO** (S3 local)
* Sistema de ficheros + estructura tipo *data lake*
* PostgreSQL (si hay datos relacionales)

#### Procesamiento

* **Spark (local o standalone)** cuando:

  * El volumen crece
  * Se combinan muchas fuentes
* Python + pandas/polars para fases iniciales

#### Formatos

* **Parquet** como formato analítico principal
* Particionado por tiempo, zona o categoría

#### Consumo

* **PIA**: pandas / scikit-learn / PyTorch / Keras
* **SBD**: DuckDB, Spark SQL, BI

### 5. Tipos de datasets especialmente adecuados para Big Data en FP

Algunos ejemplos **realistas, públicos y pedagógicos**:

#### Meteorología

* AEMET (España)
* Open-Meteo
* NOAA

Ventajas:

* Series temporales
* Datos históricos
* Enriquecimiento geográfico
* Ideal para predicción

#### Calidad del aire

* Datos abiertos de comunidades autónomas
* OpenAQ

Ventajas:

* Multifuente
* Sensores
* Datos ruidosos (veracidad)
* Relación directa con problemas reales

#### Tráfico y movilidad

* Open Data municipal
* Eventos y series temporales

#### Logs simulados

* Logs de acceso web
* Eventos de una aplicación simulada
* Ideal para ver crecimiento continuo

## Propuesta de tarea conjunta SBD + PIA

### Tarea: Construcción de un dataset real a partir de fuentes públicas

#### Contexto

Se quiere construir un dataset que permita **analizar y predecir** un fenómeno real (clima, contaminación, tráfico, etc.).

Los datos **no existen como dataset único**, sino que deben construirse a partir de **múltiples fuentes públicas**.

---

### Parte A — Sistemas de Big Data (SBD)

**Objetivo:** construir el dataset analítico.

Tareas:

1. Seleccionar **2 fuentes de datos reales** (al menos una API).
2. Diseñar el **pipeline de ingesta**.
3. Descargar datos históricos y recientes.
4. Almacenar datos en estructura tipo *data lake*:

   ```
   raw/
   processed/
   curated/
   ```
5. Transformar y unificar los datos.
6. Persistir el dataset final en **Parquet**, con particionado adecuado.
7. Documentar:

   * Fuentes
   * Frecuencia
   * Problemas de calidad
   * Decisiones técnicas

**Entregables SBD:**

* Scripts de ingesta/procesamiento
* Dataset en Parquet
* README técnico del pipeline

---

### Parte B — Programación de Inteligencia Artificial (PIA)

**Objetivo:** extraer valor del dataset construido.

Tareas:

1. Cargar el dataset generado por SBD.
2. Realizar EDA.
3. Seleccionar un problema:

   * Predicción
   * Clasificación
   * Detección de anomalías
4. Entrenar y evaluar un modelo.
5. Analizar la influencia de la calidad del dataset en el modelo.

**Entregables PIA:**

* Notebook de EDA
* Modelo entrenado
* Métricas y conclusiones

---

### Evaluación conjunta

* **SBD:** calidad del pipeline, estructura, escalabilidad.
* **PIA:** uso correcto del dataset, análisis y modelo.
* **Transversal:** coherencia entre dataset construido y problema de IA planteado.

---

### Valor pedagógico

Esta tarea permite que el alumnado entienda que:

* Big Data **no es solo Spark**.
* La IA **depende completamente de cómo se construyen los datos**.
* Un buen modelo **no compensa un mal dataset**.
* Los módulos no son compartimentos estancos.
